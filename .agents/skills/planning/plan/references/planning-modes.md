# Planning Modes

Use this reference when `plan` needs to preserve a boundary stronger than "just make a roadmap".

## Validation-First Planning

- optimize for evidence, not feature breadth
- prefer small, inspectable milestones
- include environment assumptions explicitly
- include verification commands for each phase

## Implementation-First Planning

- optimize for delivered capability
- still keep milestones bounded and verifiable

## Hybrid Planning

- separate validation milestones from implementation milestones
- include an explicit decision point before crossing from one mode to the other
