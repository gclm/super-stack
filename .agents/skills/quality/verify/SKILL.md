# Verify

Use this skill after implementation or before reporting success.

## Read First

- `.planning/REQUIREMENTS.md`
- `.planning/STATE.md`
- `protocols/verify.md`

## Goals

- gather fresh evidence
- connect evidence to the requested outcome
- report any verification gaps honestly

## Steps

1. Identify the relevant requirement or user request.
2. Choose the closest proof commands or checks.
3. Run them.
4. Summarize what is verified and what is still partial.

## Output

Report:

- verified
- partially verified
- not verified
