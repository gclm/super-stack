# Scope Alignment

Use this reference when the work may have drifted away from the original intent.

## Questions

- is this still the same task the user asked for?
- is the work still within the chosen scope mode?
- did the implementation silently expand from sample to product?
- did environment fixes or debug scaffolding become permanent product scope accidentally?

## Output Hint

Use a simple summary:

- `scope alignment: preserved`
- `scope alignment: partially drifted`
- `scope alignment: drift detected`
