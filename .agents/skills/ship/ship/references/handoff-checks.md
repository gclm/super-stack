# Handoff Checks

Use this reference when `ship` is active and you need a sharper handoff lens.

## Check Before Calling Work Ready

- latest verification evidence exists
- current output still matches intended scope
- docs and decision records are not stale
- default run or dev path still works
- commit message convention is known if a commit is expected

## Good Handoff Summary

- what is actually ready
- what is intentionally not included
- what the next operator should do next
