# Scope Modes

Use this reference when the early conversation could drift between different delivery intentions.

## Common Modes

### Validation-First

- goal is to validate skills, workflow, or architecture
- implementation should stay minimal and evidence-driven
- avoid silently drifting into full product development

### Implementation-First

- goal is to deliver real product behavior
- validation still matters, but the project is not primarily a sample

### Staged Hybrid

- first phase is validation or skeleton work
- later phases may enter product delivery
- boundaries should be explicit in `harness/state.md` or `docs/overview/roadmap.md`
- `harness/history.md` if it exists
## Capture Early

When this matters, record:

- chosen scope mode
- out-of-scope items
- doc language rules
- commit convention rules
