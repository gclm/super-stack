# 低空后端面试输出模板

在简历筛选、面试准备、候选人汇总时，优先使用这些结构，避免每次重新组织输出。

## 1. 简历筛选总表

建议列：
- 姓名
- 学历 / 当前状态
- 实习单位 / 项目背景
- 主要职责摘要
- 与岗位匹配点
- 风险点
- 推荐等级
- 建议追问方向

## 2. 候选人单人总结

建议结构：
- 候选人结论：优先面 / 可面 / 不优先
- 主要职责：用 2-4 条概括候选人真正做过的事情
- 匹配点：为什么适合低空后端
- 风险点：项目泛化、偏前端、职责不清、行业相关性弱等
- 建议追问：3-5 个最该验证的问题

## 3. 面试问题模板

### 设备/通信链路
- 你项目里的长连接或设备通信是怎么设计的？
- 如果连接断开，如何重连、补发、保证幂等？

### 任务调度与状态流转
- 任务状态机怎么设计？
- 重试、补偿、重复执行怎么控制？

### 文件/图片/视频链路
- 大文件上传、断点续传、对象存储、异步处理怎么做？
- 如果巡检图片或视频很多，后端如何削峰和治理？

### MQ 与一致性
- 如何避免重复消费？
- 消息顺序、消息丢失、堆积怎么处理？

### 工单 / 巡检闭环
- 如果让你设计一个无人机巡检任务闭环，后端会拆成哪些模块？
- 告警、工单、状态回传、报告生成之间如何衔接？

## 4. 推荐分层模板

### 优先面
- 职责直接命中：设备、实时、任务、媒体、工单、工业系统

### 可面
- Java 后端能力不错，但需要通过面试确认深度或行业迁移能力

### 不优先
- 明显偏前端、职责不清、岗位贴合度弱
