# 低空后端筛选标尺

用这份标尺判断候选人是否适合低空无人机巡检、外墙清洗、作业管理等后端岗位。

## 1. 优先信号

这些经历优先级高于单纯的技术名词堆砌：

- 设备接入、协议通信、TCP/UDP/Netty、长连接
- 实时状态回传、WebSocket 推送、链路监控
- 任务调度、定时任务、幂等、补偿机制
- 图片、视频、对象存储、文件分片上传、断点续传
- 工单、巡检、检修、MES、现场作业系统
- MQ 异步链路、削峰填谷、最终一致性
- 工业系统、能源系统、物联网、智能制造等可迁移经验

## 2. 次优信号

这些经历有价值，但岗位贴合度要结合职责深度判断：

- 通用 Java 后端项目
- 电商、秒杀、社交、本地生活平台
- RAG、知识库、AI 应用平台
- 数据平台、ETL、报表系统

如果候选人在这些项目里承担的是核心链路，也可以进入面试名单。

## 3. 降权信号

这些情况需要降级或重点核验：

- 明显偏前端、移动端、UI 交互
- 只有技术栈，没有真实职责
- 项目很多但实习经历很弱
- 主要是算法研究或论文，后端落地职责不足
- 简历文本提取质量差，关键信息无法确认

## 4. 推荐分层

### 优先面
- 实习职责与低空后端核心链路直接贴合
- 或虽非低空行业，但具备很强的设备/实时/任务/媒体/工单可迁移经验

### 可面
- Java 后端基础和项目完整度不错
- 行业相关性一般，但有一定工程落地能力
- 需要通过面试核验深度

### 不优先
- 明显偏前端或非目标方向
- 真实职责太弱
- 与岗位相关性低且替代候选人较多

## 5. 面试追问方向

对推荐候选人，优先追问：

- 设备协议和长连接如何设计
- 任务状态机、失败重试、幂等如何保证
- 图片/视频/对象存储链路如何做
- MQ 重复消费、消息丢失、顺序性怎么处理
- 工单或作业流转如何设计
- 巡检任务从下发到回传、告警、闭环的后端拆分思路
