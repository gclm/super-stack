# Scope Refresh

Use this reference when `plan` is being revisited because the user changed assumptions after work had already started.

## 1. Record The Change Explicitly

Before re-planning, capture:

- the currently active solution
- the abandoned alternative
- the new out-of-scope boundary
- what phases or tasks are affected

## 2. Design-Heavy Planning

When planning design or prototype work, record:

- artifact type
- real page coverage
- user flow coverage
- whether the task is brand exploration or product-flow exploration

## 3. Fork-Aware Planning

When the project is a fork that still needs upstream sync, record:

- product-logic areas expected to track upstream
- local brand/UI layers expected to diverge
- any merge boundary that should constrain redesign tasks
