# Conventions

## 文档语言

- 默认中文：
  - README
  - docs/
  - 方案说明
  - 阶段计划
  - 决策记录
  - 验证报告
- 代码、命令、配置键、路径、协议名保持英文

如果项目有单独语言规则，以项目规则优先。

## Git 提交规范

默认采用 Angular Commit Convention 的结构，但摘要使用中文。

格式：

```text
type(scope): 中文摘要
```

示例：

```text
feat(runtime): 增加本地运行态探测
fix(shell): 修复 fnm 初始化缺失问题
docs(plan): 补充阶段实施计划
refactor(frontend): 拆分运行时页面结构
```

## 参考项目复用边界

当项目需要“参考”已有项目时，默认优先复用：

- 信息架构
- 交互结构
- 模块边界
- 页面层次

而不是默认直接复制参考项目实现。

只有在以下条件成立时，才建议直接复用实现：

- 用户明确要求代码级复用
- 参考项目实现质量已经满足当前项目目标
- 当前项目目标是快速继承，而不是验证或重构
